    • Details

    To initiate the live encoding process, we shall trigger an AWS Lambda function in this demonstration. This function will generate encoding instances and await the injection of the live feed. When the live feed becomes available, we shall proceed with the encoding and upload the encoded output to an S3 bucket.

  • Step 2: Send a SRT live feed to Bitmovin Live Encoding Endpoint
    • Details

    Please send the SRT live stream to the ingest URL of the live feed generated by the Live Encoder. Once the cloud-based live encoding instance receives the live feed, the live encoding process will start, and the output will be uploaded to S3. Once the encoding process has started, you can monitor the progress on our dashboard (https://bitmovin.com/dashboard).

  • Step 3: Coodinating the Live to VOD manifest
    • Details

    In addition to the live encoding playback, it is also possible to utilize Live-to-VOD processing. Please select one of the available Live-to-VOD options listed below. The resulting Live-to-VOD stream will be available for playback in the designated player.

Live and Live-to-VOD Playback with Bitmovin Player

Details

It is now time to play the Live manifest, which can be done using Bitmovin Player. The left column of the player application will display the live player screen. Please clock the Load button when the stream becomes available. The right column will showcase the live-to-vod player screen, which demonstrates how we can clip the VOD manifest from Live. Please select either button which triggers an AWS Lambda function to generate a clipped VOD manifest from the Live manifest.
